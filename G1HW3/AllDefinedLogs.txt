▶server.py (서버 코드)

1. [종료 요청] {address}에서 연결 종료 요청 수신:
클라이언트가 서버에 연결 종료를 요청할 때 출력
2. [연결 종료] {address}와의 연결이 종료됨. 종료 수신 수 {exit_count}:
특정 클라이언트와의 연결이 종료되었으며, 종료 요청이 몇 번 수신되었는지 출력
3. [서버 종료] 모든 클라이언트로부터 종료 요청을 수신하여 서버를 종료합니다.:
모든 클라이언트가 종료 요청을 보내 서버가 종료될 때 출력
4. [{address}] 수신된 수식: {expression} (순번: {count}):
클라이언트로부터 특정 수식을 수신했을 때, 해당 수식과 순번을 표시
5. [경고] {address}에서 순번 {expected_count} 누락됨, 재전송 요청:
클라이언트에서 순번에 맞지 않는 데이터를 보낼 때 누락된 순번을 요청
6. 현재 {received_cnt}개 수신: 현재 서버가 수신한 수식의 총 개수를 출력
7. [{address}] 수식 대기 리스트에 추가됨: {expression}:
수신한 수식을 대기 리스트에 추가했음을 알림
8. [관리 스레드 시작] 대기 리스트에서 수식을 처리 중...:
관리 스레드가 시작되었으며, 대기 중인 수식을 처리하고 있음
9. [{address}] 계산 결과 전송: {result}:
계산을 완료한 결과를 클라이언트에게 전송할 때 출력
10. [{address}] 계산할 수식: {expression}:
대기 리스트에서 꺼낸 수식을 계산 작업을 위해 준비할 때 출력
11. [계산 스레드 시작] {calc_cnt}번 calc 생성 중...: 새로운 계산 스레드가 생성
12. [{address}] 계산 수행 완료: {expression} = {result}:
해당되는 특정 수식의 계산이 완료되고 그 결과가 표시
13. [서버 시작] {host}:{port}에서 대기 중...:
서버가 지정된 IP와 포트에서 클라이언트의 연결을 기다리고 있음
14. 클라이언트 연결 완료: {addr}: 클라이언트가 서버에 연결되었을 때 출력
15. [에러] {address}에서 수식을 수신하는 중 오류 발생: {e}
16. [에러] {address}로 결과 전송 중 오류 발생: {e}

▶client.py (클라이언트 코드)

1. [재전송] {failed_count+1}번 수식 전송: {expression}:
서버에 재전송 요청이 있을 때 해당 수식을 재전송할 때 출력
2. [전송 오류] {e}:
서버로 수식을 전송하는 중 오류가 발생 시 출력
3. [오류] 누락된 데이터 재전송 요청 (순번: {failed_count}):
서버에서 특정 순번의 데이터가 누락되었음을 알리고, 해당 순번의 데이터를 재전송 요청할 때 출력
4. [{received_cnt[0]}번 결과 수신]: {result}:
서버에서 정상적으로 수식 결과를 수신했을 때, 수신된 결과와 해당 순번을 출력
5. [수신 오류] {e}:
서버로부터 결과 수신 중 오류 발생 시 출력
6. [종료 요청] 1000개의 결과 수신 완료, 서버에 종료 요청 전송:
설정된 최대 수신 개수(MAX_RESULTS=1000)에 도달하여 서버에 연결 종료를 요청할 때 출력
7. [서버 연결] {host}:{port}에 연결됨.:
클라이언트가 서버에 처음 연결되었을 때 출력
8. [클라이언트 ID 설정] ID: {client_id}:
서버로부터 클라이언트 ID를 수신하여 ID가 설정되었음을 알릴 때 출력
9. [파일 선택] {expression_file}:
클라이언트 ID에 따라 선택된 수식 파일 경로를 출력
10. [연결 종료] 서버와의 연결이 종료됨.:
모든 수식 전송과 결과 수신이 완료되어 서버와의 연결이 종료될 때 출력
11. [{send_cnt[0]+1}번 수식 전송]: {expression}
클라이언트가 서버로 새로운 수식을 정상적으로 전송할 때, 전송한 수식의 순번과 내용을 출력
